SQL - Structured Query Language

Data

It is referred to as a collection of values that provide some information.

Entity Relationship Data Model

The Entity-Relationship (ER) model is a critical concept in structured databases, primarily used for designing and visualizing the structure of a database.

The ER model consists of three main components:

  • Entities: Objects or concepts about which data is stored
  • Attributes: Properties or characteristics of entities
  • Relationships: Associations between entities

SQL - CRUD Operations

This covers basic CRUD (Create, Read, Update, Delete) operations and JOINs

Data Definition Language (DDL)

DDL commands are used to define and modify the structure of database objects such as tables, indexes, and schemas.

  • CREATE: Creates new database objects
  • ALTER: Modifies existing database objects
  • DROP: Deletes existing database objects
  • TRUNCATE: Removes all records from a table but retains structure

Data Manipulation Language (DML)

DML commands are used for manipulating data within existing tables.

  • INSERT: Adds new records to a table
  • UPDATE: Modifies existing records
  • DELETE: Removes records from a table

Example SQL Commands

-- Create a table
CREATE TABLE Employees (
    ID INT PRIMARY KEY,
    Name VARCHAR(255),
    Age INT,
    Email VARCHAR(255)
);

-- Insert data
INSERT INTO Employees (ID, Name, Age, Email) 
VALUES (1, 'John Doe', 30, 'john@example.com');

-- Select data
SELECT * FROM Employees WHERE Age > 25;

-- Update data
UPDATE Employees SET Age = 31 WHERE ID = 1;

-- Delete data
DELETE FROM Employees WHERE ID = 1;

NoSQL - Unstructured Databases

What are Unstructured Databases?

An unstructured database refers to any system designed to store, manage, and process data that does not follow a predefined model—such as tables with rows and columns—used in relational (structured) databases.

Characteristics of Unstructured Data

  • No predefined schema: Data doesn't fit into rigid tables or fields
  • Diverse formats: May be text, images, video, audio, logs, social posts
  • Context-dependent: Meaning is embedded within the data
  • Massive volume: Often 70-90% of accumulated data is unstructured
  • Complex to index and query: Needs advanced tools for search and analysis

Types of Unstructured Databases

Database Type Description Examples
Document Stores Store JSON, XML, or similar flexible documents MongoDB, Couchbase
Key-Value Stores Store data as pairs—any object can be the value Redis, Amazon DynamoDB
Wide-Column Organize data in columns for big datasets Apache Cassandra, HBase
Object Storage Store multimedia, files, and BLOBs with metadata Amazon S3, Azure Blob

Document Databases & JSON

Document databases store data as self-contained documents, typically in JSON format:

{
  "_id": "12345",
  "name": "John Doe",
  "email": "john@example.com",
  "address": {
    "street": "123 Main St",
    "city": "Anytown",
    "zip": "12345"
  },
  "orders": [
    {"order_id": "A123", "total": 250},
    {"order_id": "B456", "total": 175}
  ]
}

MongoDB Example Operations

// Insert document
db.users.insertOne({
  name: "Jane Smith",
  age: 28,
  city: "New York"
});

// Find documents
db.users.find({age: {$gte: 25}});

// Update document
db.users.updateOne(
  {name: "Jane Smith"},
  {$set: {age: 29}}
);

// Aggregate data
db.sales.aggregate([
  {$group: {_id: "$category", total: {$sum: "$amount"}}},
  {$sort: {total: -1}}
]);

CSS - Cascading Style Sheets

What is CSS?

CSS, or Cascading Style Sheets, is a stylesheet language used in web development to control the layout and presentation of web pages. It separates content from presentation, making websites easier to maintain and update.

Importance of CSS

  • Separation of concerns: Separates content (HTML) from presentation (CSS)
  • Consistent styling: Enables consistent design across multiple pages
  • Responsive design: Allows websites to adapt to different screen sizes
  • Enhanced user experience: Creates interactive elements and animations
  • Faster loading: External CSS files can be cached by browsers

CSS Syntax

CSS syntax consists of a selector and a declaration block:

selector {
  property1: value1;
  property2: value2;
}

Types of CSS

  1. Inline CSS: Applied directly to HTML elements using the style attribute
  2. Internal CSS: Defined within the HTML document using <style> tags
  3. External CSS: Stored in separate .css files and linked to HTML

CSS Selectors

Selector Type Example Description
Element p { color: blue; } Selects all paragraph elements
Class .container { width: 80%; } Selects elements with class="container"
ID #header { background: blue; } Selects element with id="header"
Pseudo-class a:hover { color: red; } Selects links on hover

Example CSS

HTML - HyperText Markup Language

What is HTML?

HTML, or Hypertext Markup Language, is the standard markup language used for creating web pages. It defines the structure and layout of content on the internet using a series of elements represented by tags.

HTML Tag Structure

Most HTML tags consist of two parts:

  • Opening Tag: Begins the element, e.g., <html>
  • Closing Tag: Ends the element, e.g., </html>

Common HTML Tags

Tag Description
<html> Defines the root of an HTML document
<head> Contains meta-information about the document
<title> Specifies the title shown in browser's title bar
<body> Contains the visible content of the document
<h1> to <h6> Defines headings of different sizes
<p> Defines a paragraph of text
<img> Embeds images into the page
<a> Creates hyperlinks

HTML Document Structure

    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Web Page</title>
    <link rel="stylesheet" href="styles.css">

<body>
    <header>
        <h1>Welcome to My Website</h1>
        <nav>
            <ul>
                <li><a href="#home">Home</a></li>
                <li><a href="#about">About</a></li>
                <li><a href="#contact">Contact</a></li>
            </ul>
        </nav>
    </header>
    
    <main>
        <section id="home">
            <h2>Home Section</h2>
            <p>This is the main content area.</p>
            <img src="image.jpg" alt="Description" width="300">
        </section>
        
        <section id="about">
            <h2>About Us</h2>
            <p>Information about our company.</p>
        </section>
    </main>
    
    <footer>
        <p>© 2024 My Website. All rights reserved.</p>
    </footer>
</body>
</html>
